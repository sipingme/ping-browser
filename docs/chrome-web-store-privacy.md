# Chrome Web Store 发布清单

## ⚠️ 发布前必须完成的事项

### 一、隐私权规范标签页（必填）

#### 1. 单一用途说明
```
浏览器自动化工具，让 AI 代理通过本地 CLI 控制 Chrome 执行网页操作
```

#### 2. 远程代码理由
```
本扩展不加载或执行任何远程代码，所有代码都打包在扩展内
```

#### 3. 主机权限理由
```
需要在所有网站上执行自动化操作，因为用户可能需要在任意网站上使用此工具
```

#### 4. activeTab 权限理由
```
需要访问当前活动标签页以执行页面操作，如点击元素和填写表单
```

#### 5. alarms 权限理由
```
需要定时检查与本地服务的连接状态
```

#### 6. debugger 权限理由
```
需要使用 Chrome DevTools Protocol (CDP) 进行高级浏览器控制，如网络监控和性能追踪
```

#### 7. history 权限理由
```
需要访问浏览历史以提供智能推荐功能，帮助用户发现可自动化的网站
```

#### 8. scripting 权限理由
```
需要在网页中注入脚本以执行自动化操作，如获取 DOM 元素和执行用户指令
```

#### 9. storage 权限理由
```
需要存储用户配置，如上游服务地址设置
```

#### 10. tabs 权限理由
```
需要获取和管理浏览器标签页信息，以便 AI 代理能够切换、打开和关闭标签页
```

#### 11. 数据使用政策确认
- [ ] 勾选确认数据使用情况符合开发者计划政策

### 二、商品详情标签页（必填）

#### 详细说明（至少25字符）
```
ping-browser 是一个面向开发者和 AI 代理的浏览器自动化扩展。它通过本地 HTTP 服务与 CLI 工具通信，让 AI 代理能够控制您的 Chrome 浏览器执行自动化任务。

主要功能：
• 打开和导航网页
• 点击、填写表单等页面交互
• 截图和获取页面内容
• 执行自定义 JavaScript
• 管理标签页

适用场景：
• AI 辅助的网页自动化
• 开发者调试和测试
• 数据采集和处理
```

#### 屏幕截图（必须至少1张）
- 尺寸要求：1280x800 或 640x400
- 建议截图内容：扩展设置页面、CLI 运行效果
